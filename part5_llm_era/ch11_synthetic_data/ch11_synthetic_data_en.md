## §1 Theory

### 1.1 Why Is Synthetic Data Needed?

Training deep-learning ISP models (DL ISP) requires large amounts of **paired data**: noisy/degraded input images along with corresponding clean reference images. In practice, however, obtaining such real paired data faces fundamental engineering obstacles:

**Cost of collecting real paired data**:
- **Controlled laboratory scene capture**: requires an integrating sphere or professional light box to guarantee strict control of scene brightness and color temperature;
- **Multi-exposure synchronization**: the clean reference RAW requires extremely long exposure averaging or high photon count integration, and must be precisely aligned with the noisy RAW to be trained on;
- **Limited scene diversity**: the variety of scenes that can be captured in a laboratory is limited; complex conditions such as outdoor, nighttime, and motion scenes are difficult to collect in large quantities under controlled conditions;
- **Difficulty scaling across sensors**: each new sensor model requires a complete calibration dataset to be re-collected, and costs scale linearly.

SIDD (Smartphone Image Denoising Dataset; Abdelhamed et al., CVPR 2018) is the most complete real RAW noise dataset to date, capturing over 30,000 noisy/clean sRGB image pairs using 5 different smartphone models. Even so, SIDD covers only a limited set of sensor models, scene types, and ISO ranges. This limitation directly motivates the need for synthetic data.

**Core value of synthetic data**: replacing expensive real capture with parameterized physical models or generative models to produce paired training data at **near-zero collection cost** and at any scale, for any sensor model, at any noise level.

### 1.2 Forward Degradation Model

The theoretical foundation for synthetic data generation is the **forward degradation model**: given a clean image $x$, a degraded image $y$ is generated by parameterized modeling of the real imaging chain degradation process:

$$y = \mathcal{D}(x; \theta_d)$$

where $\theta_d$ is the set of degradation parameters. For ISP-related tasks, the main degradation types include:

**Gaussian noise** (Additive White Gaussian Noise, AWGN): the simplest noise model, applicable to low-ISO scenes dominated by read noise:

$$y = x + n, \quad n \sim \mathcal{N}(0, \sigma^2 \mathbf{I})$$

**Poisson shot noise**: photon counts follow a Poisson distribution; the dominant noise in high-ISO scenes:

$$y \sim \mathcal{P}(\lambda \cdot x) / \lambda, \quad \text{var}[y] = x / \lambda$$

The SNR improves as the signal increases (relatively lower noise in bright regions) — the key distinguishing feature from AWGN.

**Poisson-Gaussian noise model** (Foi et al., 2008): accounts for both shot noise and read noise simultaneously:

$$y = \mathcal{P}(\alpha \cdot x) / \alpha + \mathcal{N}(0, \beta)$$

Conditional variance:

$$\text{var}[y|x] = \frac{x}{\alpha} + \beta$$

The NLF (Noise Level Function) is thus a linear function of $x$. In the RAW domain, $\alpha$ corresponds to the reciprocal of sensor gain and $\beta$ to the variance of the read noise floor.

**JPEG compression artifacts**: blocking artifacts and ringing caused by DCT quantization, parameterized by the Quality Factor (QF).

**Motion blur**: caused by camera shake or scene motion, modeled by convolving with a blur kernel:

$$y = k * x + n$$

where $k$ is a linear motion kernel with length $L$ and orientation angle $\theta$.

**Defocus blur**: approximated with a pillbox kernel or Gaussian kernel:

$$y = \frac{1}{\pi r^2} \cdot \mathbf{1}_{||\mathbf{u}|| \leq r} * x + n$$

where $r$ is the Circle of Confusion (CoC) radius.

### 1.3 The ELD Statistical Noise Model

**ELD** (Extreme Low-light Dataset; Wei et al., CVPR 2021) proposes the most complete RAW-domain statistical noise model to date. Building on the Poisson-Gaussian foundation, it adds two noise components that previous models ignored but that are significant in real sensors:

The complete ELD noise model is:

$$y = K \cdot \mathcal{P}\!\left(\frac{x}{K}\right) + n_{\text{TL}} + n_{\text{row}} + n_{\text{FPN}}$$

Component definitions:
- $K$: sensor system gain, equal to ADC gain times analog gain;
- $K \cdot \mathcal{P}(x/K)$: gain-multiplied Poisson shot noise;
- $n_{\text{TL}} \sim \mathcal{N}(0, \sigma_{\text{TL}}^2)$: thermal noise + quantization noise, modeled as additive Gaussian;
- $n_{\text{row}}$: row noise — each row shares the same random offset, distributed as $\mathcal{N}(0, \sigma_{\text{row}}^2)$, shift-invariant in the row direction;
- $n_{\text{FPN}}$: fixed pattern noise — fixed per exposure and spatially correlated, calibrated through sensor dark frames.

The key contribution of the ELD model is its explicit modeling of **row noise** and **fixed pattern noise** — two noise types that contribute significantly in extreme low-light (very high ISO) conditions and are the root cause of many denoising method failures on datasets such as SIDD and DND.

## §2 Generation Methods

### 2.1 Degradation Pipeline Synthesis

The most classic and widely used synthesis method: manually design a sequence of degradation operators to progressively degrade a clean image into a noisy one.

**Basic degradation pipeline** (based on an inverse ISP pipeline):

```
Clean sRGB image
   → Inverse gamma (sRGB → linear RGB)
   → Inverse CCM (linear RGB → sensor RGB)
   → Inverse AWB (remove white balance gains)
   → Crop to Bayer format (RAW domain)
   → Add Poisson + Gaussian noise (ELD model)
   → Forward ISP (RAW → degraded sRGB)
   → Obtain paired (degraded sRGB, clean sRGB)
```

This method directly corresponds to the core idea of **Unprocessing** (Brooks et al., CVPR 2019): map sRGB images back to the RAW domain via inverse ISP processing, add physically correct noise in the RAW domain, then obtain degraded images via forward ISP. Since noise is added in the linear RAW domain, the physical validity of the Poisson shot noise is guaranteed (adding Poisson noise directly in the sRGB domain is physically incorrect, because the non-linear operations in the ISP alter the statistical properties of the noise).

**Higher-order degradation pipeline** (BSRGAN/Real-ESRGAN paradigm; Wang et al., ICCV 2021): proposed for super-resolution tasks, this randomly permutes the order of degradation operators at training time — randomly selecting the order of degradation operators (blur → downsampling → noise, or downsampling → blur → noise), enhancing the model's robustness to degradation order:

$$y = \mathcal{D}_{\sigma(3)} \circ \mathcal{D}_{\sigma(2)} \circ \mathcal{D}_{\sigma(1)}(x)$$

where $\sigma$ is a random permutation. This idea is equally applicable to ISP denoising/artifact-removal tasks.

### 2.2 GAN-Based Synthesis

When the degradation process is difficult to describe accurately with a parameterized physical model, a GAN (Generative Adversarial Network) can be used to learn the real degradation distribution from **unpaired data**:

**CycleGAN approach**: train two generators $G: X \to Y$ (clean → degraded) and $F: Y \to X$ (degraded → clean), and learn domain transfer without paired data through a cycle consistency loss:

$$\mathcal{L}_{\text{cyc}} = \mathbb{E}_x[\|F(G(x)) - x\|_1] + \mathbb{E}_y[\|G(F(y)) - y\|_1]$$

**CBDGAN** (Guo et al., CVPR 2019) designed a GAN framework specifically for image denoising to learn the distribution of real camera noise, significantly outperforming the simple Gaussian noise assumption on benchmarks such as SIDD.

**NoiseFlow** (Abdelhamed et al., ICCV 2019) uses normalizing flows to model the exact probability distribution of noise. Compared with the training instability of GANs, normalizing flows provide likelihood-exact noise sampling — more reliable in scenarios requiring precise probabilistic modeling.

Core advantage of GAN methods: no sensor parameter calibration is needed; the distribution is learned directly from a small number of real (noisy + clean) samples, implicitly capturing complex statistical structures that the physical model misses. Disadvantages: GAN training is unstable and may introduce mode collapse, leading to insufficient diversity in the generated noise distribution.

### 2.3 Diffusion-Based Synthesis

Diffusion models provide the latest paradigm for synthetic data generation:

**SD-XL base image generation**: first use Stable Diffusion XL to generate high-quality, diverse clean scene images (covering indoor/outdoor, low-light/bright-light, portrait/landscape, etc.) as the clean substrate for the degradation pipeline. This solves the problem of traditional methods depending on a limited library of clean images, greatly increasing the scene diversity of synthetic training data.

**Physical noise superimposition**: map the clean images generated by the diffusion model to the RAW domain via the inverse ISP pipeline, then add noise from the ELD physical noise model, ensuring the physical validity of the noise.

**IDM (Image Degradation Model)** (Luo et al., 2023) models the degradation process itself as a diffusion process, capable of introducing different types of degradation at each diffusion step to generate more realistic composite degradation samples.

**NAFNet training data** (Chen et al., ECCV 2022) adopts a mixed strategy: joint training on SIDD real paired data + large-scale synthetic degradation data, where the synthetic data compensates for SIDD's shortfall in scene diversity and the real data ensures the fidelity of noise statistics.

## §3 Domain Gap and Correction

### 3.1 Sources of Discrepancy Between Real and Synthetic Noise Statistics

Even with carefully designed physical noise models, **a statistical gap always exists between synthetic training data and real sensor data**. The fundamental sources are:

**Ignored noise correlations**: standard Poisson-Gaussian models assume spatially independent and identically distributed (i.i.d.) noise, but real sensor noise has:
- **Column correlation**: caused by offset drift in column-level amplifiers; pixels in the same column tend to shift together;
- **Block structure**: from the time-multiplexed structure of the ADC; systematic noise floor differences exist between adjacent column blocks;
- **Spatial non-uniformity of dark current**: position-dependent dark current distribution caused by sensor manufacturing defects.

**Non-Gaussian noise tails**: real noise distributions often have heavy tails — occasional hot pixels, cosmic ray hits, and other anomalies that are difficult to model with simple Gaussian/Poisson distributions.

**Non-linear propagation through the ISP**: after noise is added, the non-linear processing in the ISP (tone mapping, color matrix, etc.) transforms the statistical properties of the noise, causing the noise distribution in the final sRGB domain to deviate from the additive noise assumption.

### 3.2 Calibration-Based Domain Gap Reduction

**Noise Level Function (NLF) calibration** is the key systematic method for eliminating domain gaps:

**Calibration procedure**:
1. At a uniform-brightness scene (flat field), capture multiple frames at exposures covering the dynamic range;
2. Calculate the noise variance at each brightness level (remove fixed pattern noise by frame differencing);
3. Fit the linear model $\sigma^2(I) = \alpha I + \beta$ to obtain $(\alpha, \beta)$ at that ISO;
4. Repeat at multiple ISO values (e.g., 100, 400, 1600, 6400, 25600) to build an NLF lookup table.

The calibrated $(\alpha, \beta)$ parameters replace the default noise parameters in the synthesis pipeline, aligning the NLF curve of the synthetic data precisely with the real sensor. This step can compress the synthetic-to-real PSNR gap from a typical 1.5 dB to below 0.5 dB.

**Fixed Pattern Noise (FPN) calibration**: capture multiple dark frames (RAW with the lens completely covered), average them to obtain the FPN pattern, and compute the per-frame standard deviation to obtain the thermal noise $\sigma_{\text{TL}}$. Superimpose the calibrated FPN pattern in the synthesis pipeline.

### 3.3 Adaptive Synthesis: Domain Randomization of Noise Parameters

**Domain randomization** (Tobin et al., 2017) is an alternative path to addressing domain gap: rather than trying to precisely match a single real sensor's noise distribution, randomly sample noise parameters from their **plausible range**, forcing the model to remain robust to various noise variants:

$$\theta_d \sim \mathcal{U}[\theta_{\min}, \theta_{\max}]$$

In ISP denoising practice:
- $\alpha \sim \mathcal{U}[10^{-3}, 10^{-1}]$ (shot noise coefficient, corresponding to the ISO 100–25600 range);
- $\beta \sim \mathcal{U}[10^{-6}, 10^{-4}]$ (read noise floor variance);
- $\sigma_{\text{row}} \sim \mathcal{U}[0, 0.01]$ (row noise amplitude).

Models trained with domain randomization often generalize to new sensors outside the calibration set. The trade-off is a slightly lower peak performance on a specific sensor compared with a model trained with precisely calibrated parameters (typically a gap of 0.1–0.3 dB). Domain randomization is the more economical strategy when coverage of multiple sensor models (e.g., multi-SKU mobile platform scenarios) is required.

## §4 Sensor-Specific Synthesis

### 4.1 Mapping Synthetic Data to a Specific Sensor

Noise characteristics differ significantly across sensors. The NLF parameters of a Sony IMX989 (1-inch) and Samsung GN2 (1/1.12-inch) at the same ISO can differ by a factor of 2–3. The goal of sensor-specific synthesis is to generate synthetic training data whose statistical properties are **as close as possible to the target sensor**.

**Using sensor noise calibration databases**: some sensor manufacturers and academic institutions have published sensor noise calibration data:
- **EMVA 1288 standard**: the European Machine Vision Association's sensor noise calibration standard; NLF parameters for major sensors from Sony, Samsung, etc. can be obtained in this format;
- **DxOMark database**: SNR curves published by DxOMark can be reverse-engineered to estimate $(\alpha, \beta)$;
- **Sensor manufacturer datasheets**: typically include typical SNR performance at different ISO values, serving as a starting point for parameter estimation.

### 4.2 Synthetic Data Generation for Night-Scene Denoising on a 1-Inch Sensor

Taking the Sony IMX989 (1-inch, widely used in 2023 flagship phones) as an example for generating night-scene denoising training data:

**Target scenario**: ISO 3200–51200, exposure time 1/30s–1s, low-light indoor/nighttime street capture.

**Synthesis procedure**:
1. **Clean substrate acquisition**: use extremely long RAW exposure (1000 ms × multi-frame averaging) or a high-end DSLR to capture clean indoor/night scene reference images;
2. **NLF calibration**: calibrate $(\alpha, \beta, \sigma_{\text{row}}, \sigma_{\text{FPN}})$ for IMX989 at ISO 3200/6400/12800/25600 following the procedure in §3.2;
3. **Inverse ISP**: map the clean sRGB reference images back to IMX989's linear RAW domain (using the known CCM and white balance gains for this sensor);
4. **ELD noise superimposition**: add noise using calibrated parameters following the ELD model in §1.3;
5. **Validation**: validate NLF fitting accuracy of the synthetic data against real SIDD/ELD noise samples; ensure $\alpha$ and $\beta$ errors are < 5%.

**Recommended dataset scale**: training a denoising model at the NAFNet level requires at least 5,000 paired training samples, covering at least 4 ISO levels and 3–5 scene types.

### 4.3 Generating Universal Synthetic Data for Multi-Sensor Platforms

Phone manufacturers typically need to deploy the same DL denoising model on 5–10 different sensors; collecting real paired data per sensor is prohibitively expensive. **Multi-sensor domain randomization synthesis** strategy:

1. Calibrate NLF parameters for all target sensors, establishing parameter ranges $[\alpha_{\min}, \alpha_{\max}]$ and $[\beta_{\min}, \beta_{\max}]$;
2. Uniformly and randomly sample noise parameters from the above ranges at training time;
3. Use the sensor ID or NLF parameters themselves as additional network inputs (conditioned denoising), enabling the model to perceive the noise characteristics of the target sensor.

Conditioned denoising (blind denoising with NLF condition) can improve PSNR on each sensor by 0.3–0.8 dB over unconditioned models, and is the recommended deployment approach for multi-sensor platforms.

## §5 Evaluation

### 5.1 Statistical Fidelity Assessment

**NLF curve fitting accuracy**: the most direct evaluation metric. At the same ISO, compute $(\alpha, \beta)$ for synthetic and real data and calculate relative error:

$$\epsilon_\alpha = \frac{|\hat{\alpha} - \alpha_{\text{real}}|}{\alpha_{\text{real}}}, \quad \epsilon_\beta = \frac{|\hat{\beta} - \beta_{\text{real}}|}{\beta_{\text{real}}}$$

Typically, $\epsilon_\alpha, \epsilon_\beta < 10\%$ is required to consider the synthetic data to have sufficient physical fidelity.

**Distribution distance metrics (FID/KID)**: compute the feature distribution distance between synthetic noise patches (obtained by subtracting the mean + multi-frame-average estimated clean image) and real noise patches.

Special care is needed when using FID on the RAW domain: FID uses an Inception network pretrained on sRGB ImageNet, making it less effective for measuring RAW-domain noise. It is recommended to use **pixel-statistics-based KID** in the RAW domain (using a polynomial kernel instead of Inception features).

**Power Spectral Density (PSD) matching**: analyze the frequency components of noise patches via FFT. Real sensor column noise manifests as energy concentration in the horizontal direction in the frequency domain; a PSD comparison can visually assess whether the spatial correlations of synthetic noise are consistent with the real sensor.

### 5.2 Downstream Training Effectiveness Evaluation

The **synthetic-to-real gap** is the most critical metric for evaluating synthetic data quality:

| Training data | SIDD validation PSNR (dB) | Gap vs. real-data training |
|---|---|---|
| Real SIDD paired data | 39.7 (NAFNet) | Baseline |
| Physical parameter synthesis (uncalibrated) | 38.2 | −1.5 dB |
| Physical parameter synthesis (after NLF calibration) | 39.2 | −0.5 dB |
| ELD model synthesis | 39.4 | −0.3 dB |
| GAN synthesis (NoiseFlow) | 39.5 | −0.2 dB |

Calibrated ELD synthetic data is already very close to the training performance achievable with real data (gap < 0.3 dB), demonstrating the practical value of high-quality synthetic data.

### 5.3 NTIRE Competition Benchmarks

**NTIRE 2022 RAW Image Denoising Challenge** (Abdelhamed et al., 2022) is the authoritative benchmark for evaluating RAW denoising methods:

- Uses the extended SIDD dataset (covering more sensor models and ISO levels);
- Separate tracks evaluate training with real paired data only (Track 1) and training with additional synthetic data (Track 2);
- The Track 2 winning method (2022) used mixed training of real data + ELD synthetic data, achieving a PSNR **0.15 dB higher** than training with real data alone — demonstrating that high-quality synthetic data still provides gains on top of the strongest baseline.

### 5.4 Comprehensive Evaluation Framework

| Evaluation dimension | Specific metric | Typical passing value |
|---|---|---|
| NLF statistics | $\epsilon_\alpha$, $\epsilon_\beta$ relative error | < 10% |
| Distribution distance | KID (synthetic vs. real noise patches) | < 0.005 |
| Frequency characteristics | PSD horizontal/vertical energy ratio | < 20% deviation from real |
| Downstream effect | Synthetic training PSNR gap | < 0.5 dB |
| Scene coverage | Scene type count × ISO level count | ≥ 5 × 4 = 20 combinations |

**Module 1: Sensor NLF calibration pipeline**

```python
# Pseudocode illustration: fit NLF parameters from flat-field images
# See §6 Code section for runnable examples

def calibrate_nlf(flat_field_raws, iso_list):
    """
    Calibrate NLF parameters (alpha, beta) from multi-ISO flat-field RAW sequences.

    Args:
        flat_field_raws: dict {iso: [N, H, W] RAW frame sequence}
        iso_list: list of target ISO levels
    Returns:
        nlf_params: dict {iso: (alpha, beta)}
    """
    nlf_params = {}
    for iso in iso_list:
        frames = flat_field_raws[iso]  # [N, H, W]
        # Remove FPN by frame differencing: noise = frames[i] - mean(frames)
        mean_signal = frames.mean(axis=0)
        noise_frames = frames - mean_signal[None]

        # Estimate noise variance vs. signal intensity at different brightness regions
        signal_levels = np.linspace(mean_signal.min(), mean_signal.max(), 20)
        variances = []
        for s_level in signal_levels:
            mask = (mean_signal > s_level - 50) & (mean_signal < s_level + 50)
            if mask.sum() > 100:
                variances.append(noise_frames[:, mask].var())
            else:
                variances.append(np.nan)

        # Linear regression to fit sigma^2 = alpha * I + beta
        valid = ~np.isnan(variances)
        alpha, beta = np.polyfit(signal_levels[valid], np.array(variances)[valid], 1)
        nlf_params[iso] = (alpha, beta)

    return nlf_params
```

## §8 Glossary

| Term | Definition |
|---|---|
| **NLF** | Noise Level Function. A function describing how image noise variance varies with signal intensity, typically parameterized as the linear model $\sigma^2(I) = \alpha I + \beta$. The core tool for sensor noise calibration and synthetic data generation. |
| **Domain Randomization** | Randomly sampling degradation parameters during training (rather than fixing them to match a single real sensor) to force the model to remain robust to parameter variation. An effective low-cost strategy for covering multiple sensor models. |
| **Degradation Pipeline** | The processing chain that applies a sequence of degradation operators (noise, blur, compression, etc.) to a clean image to generate a degraded image. The most classical method for synthetic data generation. |
| **Synthetic-to-Real Gap** | The performance gap between a model trained on synthetic data and a model trained on real data, measured on a real test set. The core metric for assessing synthetic data quality. |
| **ELD** | Extreme Low-light Dataset/model. A complete RAW-domain statistical noise model proposed by Wei et al., CVPR 2021, consisting of four components: shot noise, read noise, row noise, and fixed pattern noise. |
| **FPN** | Fixed Pattern Noise. A spatially position-dependent fixed noise pattern caused by sensor pixel defects and spatial non-uniformity of dark current. Can be calibrated via dark frame capture. |
| **Unprocessing** | Inverse ISP processing. A method proposed by Brooks et al., CVPR 2019, that maps sRGB images back to the RAW linear domain so that sensor noise can be added in a physically correct domain. |
| **NR-IQA** | No-Reference Image Quality Assessment. A quality assessment method that predicts quality scores directly from degraded images without requiring a reference image. |
| **Paired Data** | (Degraded input, clean reference) image pairs required for supervised DL ISP training. The core data requirement for DL ISP research. |
| **KID** | Kernel Inception Distance. An unbiased estimator variant of FID, better suited to small-sample distribution distance evaluation. |

## §9 Mathematical Foundations of ISP Degradation Modeling

### 9.1 Precise Formulation of Physical Camera Noise Models

Section §1 introduced the Poisson-Gaussian basic framework. This section provides a more rigorous mathematical formulation to help engineers precisely match the physical process when implementing synthetic data pipelines.

The complete RAW-domain pixel readout model is:

$$y = f\!\left(x\right) \cdot g + n_{\text{read}}$$

where the symbols are defined as:
- $x$: number of photons received by the sensor pixel (expected incident photon count);
- $f(x)$: photoelectric conversion function. For an ideal linear sensor $f(x) = x$; real sensors exhibit nonlinearity (especially near full well), approximated by piecewise linear or polynomial models;
- $g$: analog gain (the main control parameter for ISO setting);
- $n_{\text{read}} \sim \mathcal{N}(0, \sigma_r^2)$: read noise, i.i.d. Gaussian with zero mean and variance $\sigma_r^2$ independent of gain $g$;
- Shot noise is implicit in the Poisson fluctuation of $x$: actual photon count $k \sim \mathcal{P}(\lambda = \alpha x)$, where $\alpha$ is the quantum efficiency coefficient; $n_{\text{shot}} = k - \alpha x$ with variance $\text{var}[n_{\text{shot}}] = \alpha x$.

Expanding the expression, the conditional variance is:

$$\text{var}[y \mid x] = g^2 \alpha x + \sigma_r^2$$

This is the complete form of the NLF (Noise Level Function): linear in $x$, with intercept (read noise) and slope (shot noise coefficient times gain squared). Engineering practice typically merges $g^2 \alpha$ into a single parameter $\hat{\alpha}$, and writes $\sigma_r^2$ as $\beta$:

$$\sigma^2(I) = \hat{\alpha} I + \beta$$

where $I = g \cdot f(x)$ is the ADC digital value corresponding to the final stored RAW data.

### 9.2 PRNU (Photo Response Non-Uniformity)

**PRNU (Photo Response Non-Uniformity)** is the fixed per-pixel gain variation caused by sensor manufacturing defects, and is the primary component of FPN:

$$y_k = f(x) \cdot g \cdot (1 + \delta_k) + n_{\text{read},k}$$

where $\delta_k$ is the PRNU coefficient of the $k$-th pixel, with zero mean and standard deviation $\sigma_{\text{PRNU}}$ typically on the order of 0.1%–1% (varying with sensor age and temperature).

PRNU characteristics:
- **Fixed pattern:** $\delta_k$ is constant across exposures (fundamentally different from random shot/read noise), not varying randomly with time;
- **Signal-dependent:** PRNU noise amplitude is proportional to signal strength $x$ (multiplicative noise), thus visible in bright regions but not in dark regions;
- **Spatial frequency:** PRNU patterns have both low-frequency (large-scale wafer non-uniformity) and high-frequency (pixel-level defects) components;
- **Calibration:** Capture a uniformly illuminated white field (flat-field correction); average multiple frames and divide by the global mean to obtain the PRNU pattern $\hat{\delta} = \bar{y} / \overline{\bar{y}} - 1$.

In high-precision synthetic data pipelines, the PRNU superposition formula is:

$$y_{\text{syn}} = y_{\text{clean}} \cdot (1 + \hat{\delta}) + n_{\text{read}}$$

For mobile sensors at standard ISO ranges, PRNU has relatively limited impact on denoising performance (PSNR difference < 0.2 dB). However, under extreme low-light conditions (ISO > 12800), where read noise is minimal, PRNU becomes the dominant fixed pattern, and ignoring PRNU causes significant visual discrepancy between synthetic and real data.

### 9.3 Real-ESRGAN's Second-Order Degradation Pipeline

Wang et al. (ICCV 2021 Workshop)'s **Second-Order Degradation Pipeline** is the current state-of-the-art for synthetic data generation in super-resolution tasks; the same design philosophy applies to ISP denoising, artifact removal, and other tasks.

**Motivation:** Real-world image degradations are typically compositions of multiple degradation types (e.g., JPEG-compressed first, then re-photographed on a screen, then screenshot-compressed again). A single-order degradation cannot cover this compound degradation space.

**Complete two-stage pipeline:**

First-order degradation (simulating "first degradation," e.g., optical/sensor degradation during capture):

$$y_1 = \downarrow_r \left[ k_1 \ast x \right] + n_1$$

where $k_1$ is a random blur kernel (Gaussian, anisotropic Gaussian, or motion kernel randomly selected), $\downarrow_r$ is random downsampling (bicubic, bilinear, or nearest neighbor randomly selected), and $n_1$ is Poisson-Gaussian noise (parameters randomly sampled from preset ranges).

Second-order degradation (simulating "second degradation," e.g., transmission/storage compression):

$$y_2 = \text{JPEG}_{\text{qf}} \left[ \downarrow_{r'} \left[ k_2 \ast y_1 \right] + n_2 \right]$$

where $k_2$, $n_2$ parameters are independently randomly sampled from the first order, and $\text{JPEG}_{\text{qf}}$ applies JPEG compression with a random quality factor $\text{qf} \sim \mathcal{U}[30, 95]$.

**Key engineering parameters (ISP denoising task example):**

| Parameter | Range | Description |
|-----------|-------|-------------|
| Blur kernel type | Isotropic Gaussian (p=0.7) / Anisotropic Gaussian (p=0.3) | Simulates lens defocus and shake |
| Gaussian kernel $\sigma$ | $[0.2, 3.0]$ | Blur strength |
| Downsampling ratio $r$ | $[1, 4]$ (integer or fractional) | Simulates insufficient resolution |
| Shot noise $\hat{\alpha}$ | $[10^{-3}, 5 \times 10^{-2}]$ | Corresponds to ISO 100–12800 |
| Read noise $\sigma_r$ | $[10^{-4}, 10^{-2}]$ | Weakly correlated with ISO |
| JPEG QF | $[30, 95]$ | Optional, task-dependent |

Through two-stage random degradation, training data covers the complete degradation space from mild (ISO 100, no blur) to severe (ISO 12800, motion blur + JPEG compression). Models trained on this data demonstrate far better generalization than single-stage degradation approaches.

## §10 Synthetic Data Quality Assessment

### 10.1 FID (Fréchet Inception Distance)

**FID (Fréchet Inception Distance; Heusel et al., NeurIPS 2017)** is the most widely used metric for assessing the distributional distance between generated and real images. It is computed using intermediate features (2048-dimensional) of a pre-trained Inception-v3 network, modeling the feature distributions of both image sets as multivariate Gaussians, and computing the Fréchet distance (Wasserstein-2 distance) between them:

$$\text{FID} = \|\mu_r - \mu_g\|^2 + \text{Tr}\!\left(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2}\right)$$

where:
- $\mu_r, \Sigma_r$: mean and covariance of real image features;
- $\mu_g, \Sigma_g$: mean and covariance of synthetic image features;
- $\text{Tr}(\cdot)$: matrix trace.

**Lower FID indicates the synthetic image distribution is closer to the real distribution** (FID = 0 means identical distributions).

**FID applications in ISP synthetic data evaluation:**
- **Evaluating noise synthesis quality:** Extract synthetic noise patches and real noise patches (from SIDD, etc.) and compute FID. Typical high-quality synthetic noise patch FID < 10; simple Gaussian noise typically > 50.
- **Evaluating degradation diversity:** FID of the synthetic degradation set relative to the real degradation set — lower means better distribution alignment.
- **Limitation:** FID uses an Inception network pre-trained on ImageNet (sRGB format), which has limited ability to perceive RAW image noise structures. Direct computation underestimates the gap. Recommendation: apply simple tone mapping to RAW noise patches before computing FID, or use KID as an alternative.

### 10.2 Perceptual Similarity (LPIPS)

**LPIPS (Learned Perceptual Image Patch Similarity; Zhang et al., CVPR 2018)** uses intermediate-layer features of a deep network (VGG16 or AlexNet) to compute the perceptual distance between two images:

$$\text{LPIPS}(x, \hat{x}) = \sum_l \frac{1}{H_l W_l} \sum_{h,w} \| w_l \odot (\phi_l(x)_{h,w} - \phi_l(\hat{x})_{h,w}) \|^2$$

where $\phi_l$ is the layer-$l$ feature and $w_l$ is the channel weight. LPIPS correlates significantly better with human perception than PSNR/SSIM (on the Berkeley Adobe Perceptual Patch Similarity dataset, LPIPS-human correlation > 0.8; PSNR ~0.6).

**LPIPS applications in synthetic data evaluation:**
1. **Synthetic-real perceptual gap for paired images:** Given the same clean image, add synthetic noise and real noise separately; compute LPIPS between the two. Ideally, the "synthetic noisy image" and "real noisy image" LPIPS should approach zero (perceptually identical noise patterns).
2. **Restoration model performance:** Denoising models trained on synthetic data are evaluated on real test data (e.g., SIDD val) using LPIPS, measuring whether synthetic data can guide the model to learn denoising priors effective for real noise.
3. **Enhancement vs. reference perceptual consistency:** For generative ISP enhancement, LPIPS quantifies the perceptual gap between output and original input, assisting in hallucination detection (LPIPS > 0.3 typically corresponds to visible hallucinations).

### 10.3 Domain Gap Quantification

The precise quantification of the **Synthetic-to-Real Gap** is the most direct metric for assessing the practical value of synthetic data:

$$\text{Gap}_{\text{PSNR}} = \text{PSNR}(\mathcal{M}_{\text{real}}) - \text{PSNR}(\mathcal{M}_{\text{syn}})$$

where $\mathcal{M}_{\text{real}}$ is trained on real paired data (e.g., SIDD training set), $\mathcal{M}_{\text{syn}}$ is trained on synthetic data, and both are evaluated on the same real test set (e.g., SIDD val).

**Typical Gap reference values (NAFNet architecture, SIDD validation set):**

| Synthetic Data Type | PSNR (SIDD val) | Gap vs. Real Data | Notes |
|--------------------|----------------|------------------|-------|
| Real SIDD training set | 39.7 dB | — | Baseline (upper bound) |
| Simple Gaussian noise | 35.8 dB | −3.9 dB | Physically unrealistic |
| Poisson-Gaussian (uncalibrated) | 38.2 dB | −1.5 dB | Basic physical model |
| Poisson-Gaussian (NLF calibrated) | 39.2 dB | −0.5 dB | Significant improvement after calibration |
| ELD full model (with row noise + FPN) | 39.4 dB | −0.3 dB | Near real-data performance |
| GAN synthesis (NoiseFlow) | 39.5 dB | −0.2 dB | Learned distribution, closest |

**SIDD_val / DND as real distribution standards:**
- **SIDD validation set:** 1280 paired images (5 smartphone models, multiple ISO levels) — the standard test benchmark for mobile ISP denoising; not for training;
- **DND (Darmstadt Noise Dataset; Plötz & Roth, CVPR 2017):** 50 high-quality DSLR-captured noisy images; only noisy images provided (no paired clean), scored via online evaluation server — the "blind test" standard for evaluating real-world generalization.

Reporting both SIDD val (with reference, PSNR/SSIM) and DND (no reference, PSNR from online server) results is the most complete validation approach for synthetic data evaluation.

## §11 High-Quality Synthetic Data Generation Methods

### 11.1 CycleISP (CVPR 2020)

**CycleISP (Zamir et al., CVPR 2020)** is a cycle-consistent framework specifically designed for ISP training data synthesis. The core innovation is simultaneously learning invertible mappings in both **sRGB→RAW** and **RAW→sRGB** directions:

**Network structure:**

$$G_{\text{sRGB→RAW}}: \quad \hat{r} = G_1(s)$$
$$G_{\text{RAW→sRGB}}: \quad \hat{s} = G_2(r)$$

Cycle consistency loss:

$$\mathcal{L}_{\text{cyc}} = \|G_2(G_1(s)) - s\|_1 + \|G_1(G_2(r)) - r\|_1$$

CycleISP also introduces a **Noise Injection Module (NIM):** physical noise (Poisson + Gaussian) is inserted in the RAW domain after sRGB→RAW mapping but before RAW→sRGB mapping, giving the generated degraded image a physically plausible noise distribution:

$$r_{\text{noisy}} = G_1(s) + n_{\text{phy}}, \quad n_{\text{phy}} \sim \mathcal{P}\text{-}\mathcal{G}(\hat{\alpha}, \hat{\beta})$$

**CycleISP practical results:**
- NAFNet trained on synthesized (noisy sRGB, clean sRGB) pairs reaches SIDD val PSNR 39.3 dB (close to 39.7 dB from real-data training);
- Particularly suitable for scenarios lacking camera NLF calibration data: CycleISP learns sensor characteristics from a small number of real (unpaired) RAW images;
- Code: https://github.com/swz30/CycleISP

### 11.2 NoiseFlow (ICCV 2019)

**NoiseFlow (Abdelhamed et al., ICCV 2019)** uses a **Conditional Normalizing Flow** to precisely model the probability distribution of RAW-domain noise:

Given a clean signal $x$, NoiseFlow learns the conditional distribution $p(n|x)$ (distribution of noise given the signal), where $n = y - x$:

$$\log p(n|x; \theta) = \log p_z(f_\theta(n; x)) + \log \left|\det \frac{\partial f_\theta}{\partial n}\right|$$

The normalizing flow $f_\theta$ maps the complex noise distribution to standard Gaussian; trained by maximizing log-likelihood. At inference, sample $z \sim \mathcal{N}(0, I)$ and apply the inverse transform to obtain real-distribution noise.

**Key advantage:** Normalizing flows provide **likelihood-exact** noise sampling (unlike GAN's implicit distribution learning). On SIDD dataset noise fitting NLL (negative log-likelihood), NoiseFlow is ~25% lower than simple Gaussian models, capturing non-Gaussian properties of real noise (heavy tails, spatial correlations) more precisely.

### 11.3 Sensor-Parameterized Noise Generation

Beyond data-driven learning methods, direct generation based on sensor physical parameters is widely used in industry — especially suited for rapidly establishing noise models for new sensors during mass production.

**Key physical parameters and their noise correspondence:**

| Parameter | Physical Meaning | Impact on $(\hat{\alpha}, \beta)$ |
|-----------|-----------------|----------------------------------|
| **ISO (sensitivity)** | Analog gain $g = g_0 \cdot \text{ISO} / \text{ISO}_{\text{base}}$ | $\hat{\alpha} \propto g^2$, $\beta \propto g^2 \sigma_r^2$ |
| **Temperature (°C)** | Dark current increases exponentially (~2× per 8–10°C) | $\beta$ increases (read noise floor rises) |
| **Exposure time (s)** | Thermal carrier integration: $n_{\text{dark}} \propto t_{\text{exp}}$ | Dark current significant at very long exposures (> 1s); negligible for normal exposures |
| **Sensor size** | Larger pixels → more photon full-well → lower $\hat{\alpha}$ / better SNR | 1-inch sensor $\hat{\alpha}$ is ~1/4–1/8 that of 1/2.5-inch |
| **Bayer color channel** | G channel dual-sampled (two green pixels), R/B single-sampled | G channel effective $\hat{\alpha}$ is 1/2 of R/B |

**Industry practice: Temperature-ISO 2D noise model**

Precision industrial noise models build a 2D lookup table over (ISO, temperature):

$$(\hat{\alpha}, \beta) = \text{LUT}[\text{ISO}, T_{\text{sensor}}]$$

For example, the Sony IMX989 at ISO 3200, 25°C has $\hat{\alpha} \approx 2.1 \times 10^{-4}$; at ISO 3200, 50°C (high temperature environment), $\hat{\alpha} \approx 2.3 \times 10^{-4}$, with $\beta$ increasing ~40%. Precise temperature compensation makes synthetic data for extreme low-light night photography more realistic (phone sensors can rise 15–25°C due to heat from continuous shooting).

## §12 Data Augmentation Strategies

### 12.1 ISP-Task-Specific Augmentation

Standard data augmentation strategies for image classification (random horizontal flip, random crop, color jitter) must be customized for ISP tasks according to the physical characteristics of noise, to avoid distorting noise statistics.

**Random Crop:**

When cropping RAW images, **Bayer alignment must be maintained** (2×2 block alignment). If the crop position is not at even coordinates, the RGGB grid pattern is disrupted, causing systematic color errors in demosaicing output. Implementation must round the crop origin to $2 \times \lfloor h/2 \rfloor$ and $2 \times \lfloor w/2 \rfloor$.

**Random Flip:**

Horizontal and vertical flips affect Bayer format differently:
- Horizontal flip: RGGB becomes GRRG (G and R swapped), requiring post-flip Bayer realignment (1-pixel shift) or corresponding adjustment of the demosaicing input format assumption;
- Vertical flip: RGGB becomes GBRG, similarly requiring handling.
- Practical approach: Apply flips in the sRGB domain (after ISP) to avoid RAW-domain Bayer alignment issues; if flipping in RAW domain, crop a 1-pixel border after flipping to restore Bayer format.

**Impact of flips on spatial noise distribution:**

Row noise appears as horizontal stripes in RAW images — each row of pixels shares the same random offset. Vertical flips do not change row noise statistics (rows remain rows after flipping). Horizontal flips convert row noise into "column noise," changing the anisotropic character of the noise, which may slightly degrade row noise removal performance. For denoising tasks, it is recommended to use only vertical flips (not horizontal) to preserve row noise directionality.

### 12.2 Mixup for Color Augmentation

**Mixup (Zhang et al., ICLR 2018)** linearly interpolates inputs and labels between two training samples:

$$\tilde{x} = \lambda x_i + (1-\lambda) x_j, \quad \tilde{y} = \lambda y_i + (1-\lambda) y_j, \quad \lambda \sim \text{Beta}(\alpha, \alpha)$$

In ISP color enhancement tasks (CCM learning, color transfer), Mixup can generate smooth interpolated samples in color space, improving model robustness to intermediate color tones. However, note: **for noise-related ISP tasks (denoising), Mixup in the RAW domain destroys the statistical properties of Poisson shot noise** — the linear combination of two noisy images is not equivalent to a single noisy image with combined brightness. Therefore, Mixup is only appropriate in the sRGB domain for color/quality evaluation tasks, not for RAW-domain denoising.

### 12.3 Core Differences: RAW Domain vs. sRGB Domain Augmentation

| Dimension | RAW Domain Augmentation | sRGB Domain Augmentation |
|-----------|------------------------|--------------------------|
| **Noise physicality** | High (Poisson-Gaussian noise parametrically added, physically correct) | Low (sRGB-domain noise non-linearly processed by ISP, difficult to model precisely) |
| **Bayer format constraint** | Must maintain Bayer alignment (higher augmentation complexity) | No constraint (any augmentation applicable) |
| **Color augmentation flexibility** | Limited (before CCM, color space is linear RGB, not suited for perceptual color transforms) | High (sRGB/Lab space more intuitive, color jitter straightforward) |
| **Applicable tasks** | Denoising, demosaicing, RAW SR | Tone mapping, color enhancement, perceptual IQA |
| **Training efficiency** | Lower (augmentation logic is complex) | High (standard library support, GPU-acceleratable) |
| **End-to-end ISP learning** | Strongly recommended (avoids ISP-introduced information loss) | Not applicable (no access to RAW intermediate state) |

**Industry practice recommendation:** For ISP models with RAW input (RAW denoising, RAW SR, RAW→sRGB end-to-end ISP), all noise synthesis should be performed in the RAW domain; sRGB domain is limited to lightweight geometric augmentations (horizontal flip, random rotation). For tasks with sRGB input (sRGB SR, compression artifact removal, perceptual IQA), use standard augmentation libraries in the sRGB domain (albumentations, torchvision.transforms).

### 12.4 AutoAugment/RandAugment for IQA Tasks

**AutoAugment (Cubuk et al., CVPR 2019)** uses reinforcement learning to search for optimal augmentation strategy combinations. **RandAugment (Cubuk et al., NeurIPS 2020)** simplifies the search using uniform random sampling and a unified magnitude parameter.

In ISP-related IQA (Image Quality Assessment) tasks, RandAugment has been shown to be effective:
- **NIQE/BRISQUE training augmentation:** Randomly applying mild degradations (blur, noise, brightness changes) to clean images increases IQA model sensitivity to various quality issues, improving SRCC (Spearman Rank Correlation Coefficient) by ~0.03–0.05;
- **Subjective IQA dataset expansion:** Human-rated datasets are typically small (e.g., TID2013 has only 3000 images). RandAugment can expand training samples without changing relative quality ordering (rotation, flips don't affect quality perception; brightness/color transforms require caution);
- **Contrastive learning-based IQA:** Treat two augmented versions of the same image as positive pairs, different images as negative pairs, and train a feature encoder with contrastive loss (approaches used by CONTRIQUE, CLIP-IQA, etc.).

**Limitation:** AutoAugment/RandAugment were originally designed for classification tasks (pixel-level fidelity irrelevant). Direct application in ISP restoration tasks (denoising, SR) introduces unnecessary degradations; augmentation operations must be strictly filtered (keeping only geometric transforms that don't introduce new degradations).

## §13 Synthetic Data in Industrial Practice

### 13.1 Empirical Observations on Industrial Data Mixing Ratios

Composition of DL ISP model training data at mobile manufacturers (based on public technical reports and industry exchanges — approximate estimates; actual ratios vary significantly by task and manufacturer):

**Denoising:**

| Manufacturer (representative) | Synthetic ratio | Real ratio | Synthetic data source |
|------------------------------|----------------|-----------|----------------------|
| **Xiaomi (representative practice)** | ~70% | ~30% | ELD model + sensor calibration |
| **Huawei (representative practice)** | ~60% | ~40% | GAN synthesis + physical model hybrid |
| **Google Pixel (published papers)** | ~65% | ~35% | Unprocessing method |

**Super-Resolution:**

SR tasks use an even higher proportion of synthetic data because "real paired data is difficult to collect" (requires a high-resolution reference of the same scene):
- Real-ESRGAN and similar methods use **100% synthetic data** (real HD images + second-order degradation pipeline);
- Some manufacturers supplement with a small amount (~10%) of real scene-collected (phone image, DSLR reference) pairs to calibrate final output color.

**Data mixing design principles:**
1. **Synthetic data covers edge cases:** Real-world collected data tends to concentrate in common ISO ranges (400–3200); extremely high ISO (> 12800) and extreme degradation scenarios are scarce; synthetic data can supply unlimited edge cases;
2. **Real data anchors the distribution:** Even a small amount of real paired data (as few as 500–1000 pairs) is critical for "anchoring" the model to the real sensor distribution and preventing overfitting on synthetic data;
3. **Scene balance:** Avoid imbalanced scene distribution (indoor/outdoor, day/night, portrait/landscape); the source of clean images for synthetic data must cover the target application scenarios.

### 13.2 Synthetic-Data-Driven Closed-Loop Optimization (Data Flywheel)

The most advanced ISP training systems do not generate synthetic data once but build a **continuously self-improving closed-loop system (Data Flywheel)**:

```
┌─────────────────────────────────────────────────────┐
│                 Data Flywheel Loop                   │
│                                                     │
│  ① Simulator                                        │
│     Physical noise model + degradation pipeline     │
│     ↓ Generate large volumes of synthetic training data │
│                                                     │
│  ② Model Training                                   │
│     Train ISP model on synthetic data               │
│     ↓ Produce candidate model v_n                   │
│                                                     │
│  ③ Automated Evaluation                             │
│     Deploy to test phone, real scene capture        │
│     NIQE + manual annotation of small sample        │
│     ↓ Identify simulator weaknesses                 │
│       (which real scenes are poorly simulated)      │
│                                                     │
│  ④ Simulator Update                                 │
│     Adjust noise parameters and degradation         │
│     distribution based on evaluation feedback       │
│     ↓ Return to step ①                             │
│                                                     │
│  Each iteration cycle: 2–4 weeks                    │
└─────────────────────────────────────────────────────┘
```

**Core value of the closed loop:**
- **Automatically discovers simulation blind spots:** When real capture shows significantly worse performance in a specific scene type (e.g., 3200K color temperature + ISO 6400 night scene) vs. simulation tests, that scene type's degradation characteristics are not covered by the simulator;
- **Iteratively improves the simulator:** Each generation of the simulator is more precise in the scenes where the previous generation failed; model performance improves continuously;
- **Reduces real data collection:** As simulator accuracy improves, dependence on large-scale real paired data collection decreases.

**Typical Data Flywheel gains:** After 3–4 simulator optimization iterations (~3 months), PSNR in extreme low-light scenes (ISO > 12800) typically improves 0.5–1.0 dB, manual evaluation satisfaction improves 10–15%, with no increase in real data collection.

### 13.3 Large-Scale Synthetic Data Engineering Infrastructure

Large-scale synthetic data generation is not only an algorithmic problem but also an engineering scalability problem:

**Storage and format:**
- Synthetic RAW data is typically stored in **HDF5** or **LMDB** format, supporting random access (high-frequency random access by DataLoader during training);
- A single 512×512 RAW paired sample (uint16) is ~2 MB; 1 million training pairs is ~2 TB, requiring distributed storage;
- **On-the-fly synthesis** is a more flexible alternative: generate degraded images from clean images in real time each training batch, eliminating the need to pre-store degraded data, at the cost of CPU compute overhead (typically managed to maintain GPU utilization > 80%).

**Distributed synthesis acceleration:**
- Noise synthesis (Poisson sampling + Gaussian addition) is highly parallelizable; implementing custom PyTorch noise sampling operators in CUDA is 50–100× faster than CPU synthesis;
- For large-scale synthesis (e.g., 1 million training pairs), use distributed frameworks such as Ray or Dask to parallelize generation on compute clusters. Typical speed: 100 A100 GPUs in parallel, < 4 hours to generate 1 million pairs.

**Quality sampling inspection:**
- Each batch of synthetic data must automatically verify NLF parameter alignment (relative error vs. target sensor < 10%);
- Periodically randomly sample from synthetic datasets for manual reviewer confirmation of noise appearance realism (at least 100 pairs inspected weekly);
- On detecting anomalies (e.g., block artifacts in synthetic noise, NLF drift), immediately trigger simulator rollback and regeneration.

## §15 Glossary Extension

| Term | Definition |
|------|-----------|
| **PRNU** | Photo Response Non-Uniformity. Fixed per-pixel gain variation from sensor manufacturing defects. Multiplicative noise proportional to signal; the main component of FPN at standard ISP operating conditions. |
| **Second-Order Degradation** | Real-ESRGAN's cascaded two-stage degradation pipeline. First stage applies random blur + downsampling + noise; second stage applies a second independent round of blur + noise + JPEG compression. Covers compound degradation spaces that single-stage pipelines miss. |
| **Data Flywheel** | A self-improving closed-loop system for ISP training data: synthetic data generation → model training → real-world evaluation → simulator update → repeat. The standard practice in production-level mobile ISP development. |
| **On-the-fly Synthesis** | Generating degraded training images from clean images in real time each training batch, rather than pre-generating and storing. Avoids storage costs at the expense of CPU overhead. |
| **SIDD** | Smartphone Image Denoising Dataset. ~30,000 paired noisy/clean image pairs from 5 smartphone models. The standard benchmark for mobile ISP denoising evaluation (test set not for training). |
| **DND** | Darmstadt Noise Dataset. 50 DSLR noisy images (no clean reference); evaluated via online server. The "blind test" standard for real-world denoising generalization. |
| **NLF calibration** | Fitting the Noise Level Function $\sigma^2(I) = \hat{\alpha} I + \beta$ to a specific sensor by capturing calibration frames at multiple ISO levels and fitting the slope and intercept via least squares. |
