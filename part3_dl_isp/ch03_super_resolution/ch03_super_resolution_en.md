## §1 原理 (Theory)

### The Super-Resolution Problem

Image super-resolution (SR) is the task of reconstructing a high-resolution (HR) image of spatial dimensions H × W × C from a low-resolution (LR) input of dimensions (H/s) × (W/s) × C, where s is the upscaling factor. The problem is fundamentally ill-posed: many HR images can be consistently downsampled to produce the same LR observation, so the SR network must learn to hallucinate plausible high-frequency detail.

Formally, the degradation model relates HR image **x** to LR observation **y** as:

```
y = D(x; δ)
```

where D is a degradation operator parameterized by δ. The SR network f_θ learns an inverse mapping:

```
x̂ = f_θ(y)  ≈  x
```

Training minimizes a loss L(x̂, x) over a dataset of (LR, HR) pairs generated by applying D to collected HR images.

### Degradation Models

The choice of degradation model critically determines what artifacts the trained network can handle.

**Bicubic degradation.** The classical and most widely studied degradation. The HR image is convolved with a bicubic kernel and then subsampled by factor s. This produces LR images that are blurry but free of sensor noise, compression artifacts, or other real-world corruptions. Models trained on bicubic LR images achieve high PSNR scores on synthetic benchmarks but often fail on real-world photographs.

**Real-world degradation.** Real camera images suffer from a cascade of degradation sources: optical blur from the lens PSF, sensor read noise and shot noise, JPEG compression during storage, re-sharpening and other in-camera processing, and sometimes additional re-compression when images are shared online. The Real-ESRGAN degradation pipeline (Wang et al., 2021) models this cascade explicitly:

```
y = JPEG( downsample( noise( blur( x ) ) ) )
```

The pipeline applies a randomly sampled sequence of: isotropic or anisotropic Gaussian blur, sinc ringing kernels, bicubic / bilinear / nearest-neighbour downsampling, Gaussian / Poisson noise, and JPEG compression at random quality factors. A second pass of blur-noise-JPEG is applied to handle double-compression. Training on this diverse degradation space produces models that generalize to a wide range of real photographs.

### SRCNN: The First CNN Super-Resolution Network

Dong et al. (2014) introduced SRCNN, the first end-to-end convolutional network for SR. The network is remarkably simple, consisting of three convolutional layers:

1. **Patch extraction and representation:** A 9×9 conv with 64 filters extracts overlapping patches from the (bicubic pre-upsampled) LR image.
2. **Nonlinear mapping:** A 1×1 conv with 32 filters maps the patch representations to HR patch representations.
3. **Reconstruction:** A 5×5 conv with C output channels aggregates the HR patches into the final image.

SRCNN is trained with per-pixel L2 loss (mean squared error). Despite its simplicity, it outperformed all classical SR methods of its time. Its weakness is that the pre-upsampling step is performed by bicubic interpolation before the network, so the network does not learn to upsample — it learns only to sharpen.

### ESRGAN: Enhanced Super-Resolution GAN

Wang et al. (2018) proposed ESRGAN, which introduced two key innovations over the earlier SRGAN:

**Residual-in-Residual Dense Block (RRDB).** The generator G uses RRDBs as its core building block. Each RRDB consists of three densely-connected convolutional layers (dense block) nested inside a residual skip connection. A learnable scaling factor β (default 0.2) multiplies the residual before addition to prevent training instability:

```
RRDB output = x + β · DenseBlock3(DenseBlock2(DenseBlock1(x)))
```

Stacking multiple RRDBs (typically 23) provides very deep feature extraction with stable gradients. Batch normalization is removed compared to SRGAN because it introduces artifacts when the model is trained on diverse texture content.

**Training objective.** ESRGAN uses a three-term loss:

```
L_G = L_pixel + λ_perc · L_perceptual + λ_GAN · L_GAN
```

- **L_pixel:** L1 pixel loss between generated and HR image (provides stable initial training).
- **L_perceptual:** VGG feature-space distance, computed on activations before ReLU rather than after (as in SRGAN). This preserves stronger gradient signal and produces sharper textures.
- **L_GAN:** Relativistic average GAN loss. The discriminator D_Ra predicts whether a real image is more realistic *on average* than fake images, rather than classifying each image independently. This makes the discriminator's task harder and the generator's perceptual output sharper.

**Two-stage training.** ESRGAN is first pre-trained with only L_pixel (called RRDB_PSNR), then fine-tuned with all three loss terms. This avoids mode collapse during early GAN training.

### Real-ESRGAN

Real-ESRGAN (Wang et al., 2021) keeps the ESRGAN generator architecture but replaces the bicubic degradation pipeline with the multi-order random degradation described above. It also uses a U-Net discriminator with spectral normalization to better capture local texture artifacts. The discriminator receives both the generator's output and randomly sampled real-world photographs to learn a broader definition of "realistic."

### SwinIR: Transformer-based Image Restoration

Liang et al. (2021) replaced convolutional feature extraction with Swin Transformer blocks. SwinIR consists of three stages:

1. **Shallow feature extraction:** A single conv layer maps the input to feature space.
2. **Deep feature extraction:** Multiple Residual Swin Transformer Blocks (RSTB), each containing several Swin Transformer layers with window-based self-attention plus a residual convolutional layer.
3. **Image reconstruction:** Pixel-shuffle upsampling followed by a conv to produce the SR output.

Window-based self-attention (Shifted Window, SW-MSA) allows SwinIR to capture long-range dependencies within local windows while remaining computationally tractable. The window size (typically 8×8) can be shifted between consecutive layers so information flows across window boundaries.

SwinIR achieves state-of-the-art PSNR on classical benchmarks while also producing excellent perceptual quality when trained with GAN loss.

### The Perceptual-Distortion Trade-off

Blau and Michaeli (2018) proved a fundamental theoretical result: there exists a trade-off between perceptual quality (measured by the distance between the distribution of SR outputs and the distribution of natural HR images) and distortion (measured by PSNR or SSIM against a specific ground-truth HR image). Any SR algorithm can only improve one metric at the expense of the other. This explains why GAN-based methods score lower PSNR than their PSNR-optimized counterparts despite appearing visually sharper — they are trading pixel fidelity for distributional realism.

## §2 标定 (Calibration)

### Degradation Model Calibration

When deploying SR in a real ISP pipeline, the degradation model should match the actual sensor and processing chain:

- **Measure the camera's PSF** at the target resolution using a slanted-edge or point-spread function target. Fit a Gaussian (or sum of Gaussians) to characterize the optical blur.
- **Measure noise levels** at each ISO setting using flat-field captures. Fit a Poisson-Gaussian noise model: σ²(I) = a·I + b, where a is the shot noise coefficient and b is the read noise floor.
- **Capture real LR/HR pairs** using optical zoom (RealSR dataset methodology): capture the same scene with full zoom (HR) and no zoom (LR), then align using homography. This produces paired data that truly reflects the camera's degradation.
- **Iterative refinement:** Train on calibrated synthetic pairs, evaluate on real captures, and adjust the degradation parameters if the perceptual gap is large.

### Test Datasets

## §3 调参 (Tuning)

### Scale Factor Selection

The scale factor s determines the difficulty of the SR task and the required network capacity:

- **2× SR:** Moderate difficulty. Most networks achieve >34 dB PSNR on Set5. Suitable for moderate bandwidth savings.
- **4× SR:** The standard benchmark. Networks must reconstruct from 1/16 of original pixels. This is the sweet spot for most ISP applications.
- **8× SR:** Extremely ill-posed. PSNR drops significantly and hallucination risk is high. Only recommended when the LR capture is a hard constraint.

For ISP applications, 2× is often the practical choice: it allows a lower-resolution sensor to produce output equivalent to a higher-resolution sensor, with a manageable network size and inference time.

### Perceptual Weight λ

The balance between pixel loss and perceptual/GAN loss is the primary tuning knob:

- **λ_perc = 0:** Pure pixel loss. Maximum PSNR, blurry output with "watercolor" texture.
- **λ_perc = 0.1–1.0:** Balanced regime. Slight PSNR decrease, noticeably sharper textures.
- **λ_perc > 5.0:** Strong perceptual mode. PSNR decreases further, very sharp but may introduce ringing or noise amplification in flat regions.

Tune λ_perc by visual inspection on device-captured test images, not just on synthetic benchmarks.

## §4 Artifacts

### Checkerboard Artifacts

**Cause:** Transposed convolution (also called deconvolution) used for upsampling introduces a periodic pattern when the stride does not evenly divide the kernel size, creating unequal gradient accumulation across output pixels.

**Fix:** Replace transposed convolution with pixel-shuffle (sub-pixel convolution). Pixel-shuffle rearranges a (H × W × s²C) feature map into (sH × sW × C) by periodic reshuffling, producing uniform upsampling without periodic artifacts.

### Over-sharpening and Noise Amplification

**Cause:** A high perceptual loss weight encourages the generator to produce high-frequency content. In regions with low signal-to-noise ratio (dark areas, smooth sky regions), the network amplifies noise as if it were texture.

**Fix:** Use spatially adaptive weighting: lower the perceptual weight in regions classified as smooth (detected by local variance). Alternatively, apply a mild post-processing step (guided filter or bilateral filter) on smooth regions only.

### Hallucination

**Cause:** GAN-based SR generates plausible-looking texture that was not present in the original HR image. Text, license plates, and facial features are particularly susceptible. The network fills in high-frequency detail based on learned priors, not actual scene content.

**Fix:** For applications where texture fidelity is critical (document SR, forensics), use PSNR-optimized models (λ_perc = 0) and accept blurry output. For general photography, hallucination is often perceptually acceptable.

### Scale Mismatch

**Cause:** A network trained at scale ×4 is applied to an image that was actually downsampled at ×3, or vice versa.

**Fix:** Use the correct scale at inference. If the actual scale is unknown, estimate it from the LR/HR pair statistics (blur kernel estimation) or design a scale-adaptive network.

## §5 评测 (Evaluation)

### Fidelity Metrics

**PSNR (Peak Signal-to-Noise Ratio):** Measures pixel-level fidelity in dB. Higher is better. Standard threshold for "good quality" is >30 dB, though perceptual quality can be high at lower PSNR.

```
PSNR = 10 · log10(MAX² / MSE)
```

**SSIM (Structural Similarity Index):** Measures similarity in terms of luminance, contrast, and structure. Range [0, 1], higher is better. More correlated with human perception than PSNR, but still a fidelity metric that penalizes sharp GAN outputs.

### Perceptual Metrics

**LPIPS (Learned Perceptual Image Patch Similarity):** Computes distance between deep feature activations (VGG or AlexNet) of the generated and reference image. Lower is better. LPIPS correlates better with human judgments of perceptual quality than PSNR or SSIM.

### Subjective Evaluation

**MOS (Mean Opinion Score):** Human raters score each image on a Likert scale (typically 1–5). The gold standard for perceptual quality but expensive and non-reproducible across studies.

### The Anti-correlation of PSNR and LPIPS

For SR, PSNR and LPIPS are often anti-correlated: GAN-based methods with high perceptual quality score lower PSNR because they hallucinate textures that differ pixel-by-pixel from ground truth. When reporting results, always include both a fidelity metric (PSNR/SSIM) and a perceptual metric (LPIPS/MOS) to give a complete picture.

## §7 Checkerboard Artifacts and Upsampling Method Comparison

### 7.1 Root Cause of Checkerboard Artifacts

- **Cause:** Transposed convolution (deconvolution) with stride > 1 generates each output pixel from non-overlapping kernel regions, producing a periodic checkerboard pattern in the output.
- **Frequency-domain interpretation:** Spectral aliasing appears at multiples of $k \cdot f_s / \text{stride}$, visible to the naked eye as a checkerboard texture.
- **Fundamental fix:** Avoid using transposed convolution for upsampling.

### 7.2 Comprehensive Upsampling Method Comparison

| Method | Checkerboard | Compute | Detail Retention | Typical Use |
|--------|-------------|---------|-----------------|-------------|
| Bilinear interpolation + Conv | None | Low | Fair | Lightweight model preprocessing |
| Pixel-Shuffle (PixelShuffle) | None | Medium | Good | Standard in ESRGAN/SwinIR |
| Transposed convolution | Present | Medium | Good | Requires checkerboard loss to suppress |
| Bicubic interpolation + Conv | None | Medium-high | Good | High-quality post-processing |
| CARAFE (content-aware) | None | High | Best | Flagship SR models |

```python
# PixelShuffle upsampling (recommended — no checkerboard artifacts)
import torch.nn as nn

class UpsampleBlock(nn.Module):
    def __init__(self, in_ch, scale):
        super().__init__()
        self.conv = nn.Conv2d(in_ch, in_ch * scale**2, 3, padding=1)
        self.shuffle = nn.PixelShuffle(scale)  # (B, C*r², H, W) → (B, C, rH, rW)
    def forward(self, x):
        return self.shuffle(self.conv(x))
```

### 7.3 Compute Cost vs. Quality Trade-offs

- The complexity ratio between 2× and 4× SR is approximately 1:4 (receptive field and parameter count scale quadratically with the factor).
- Reference measurements on Dimensity 9300 (Imagiq 990 + APU):
  - ESRGAN 2× (640×480 → 1280×960): ~45 ms (INT8)
  - SwinIR 4× (360×240 → 1440×960): ~180 ms (INT8)

## §8 Glossary

**Super-Resolution (SR)**
The inverse problem of reconstructing a high-resolution output $H \times W$ from a low-resolution input $(H/s) \times (W/s)$, where the upscaling factor $s$ is typically 2, 4, or 8. The problem is fundamentally ill-posed: a single LR image corresponds to infinitely many plausible HR images, so the network must learn high-frequency priors. Training data is generated via a degradation model $y = D(x; \delta)$.

**Degradation Model**
The operator $D$ that maps an HR image to an LR observation. Classical degradation: bicubic downsampling. Real-world degradation (Real-ESRGAN): cascaded blur → noise → downsample → JPEG, with random parameters sampled multiple times to simulate diverse real-world conditions. The fidelity of the degradation model directly determines how well the trained network generalizes to real photographs.

**SRCNN**
The first end-to-end SR convolutional network (Dong et al., ECCV 2014) [1], with only three layers: 9×9 feature extraction → 1×1 nonlinear mapping → 5×5 reconstruction, ~8,032 parameters. Trained with MSE (L2) loss to maximize PSNR. Weakness: upsampling is performed by bicubic interpolation before the network; the network only sharpens. Set5 ×4 PSNR: 30.48 dB [1].

**RRDB (Residual-in-Residual Dense Block)**
The core building block of ESRGAN: three densely-connected conv layers nested inside a residual skip connection. Key detail: a **fixed** residual scaling factor $\beta = 0.2$ (not a learned parameter) multiplies each residual before addition to stabilize training in very deep networks (23 stacked RRDBs). Batch normalization is removed to avoid artifacts when training on diverse texture content.

**ESRGAN (Enhanced Super-Resolution GAN)**
Wang et al., ECCV Workshops 2018 [2]. Core innovations: Relativistic average GAN loss (the discriminator predicts "whether real images are more realistic on average than fake images") and VGG pre-activation perceptual loss. Two-stage training: first L1-only pre-training (RRDB-PSNR), then GAN fine-tuning. Set5 ×4 PSNR ~30.46 dB [2] (the PSNR cost of perceptual optimization), but visual texture sharpness far exceeds PSNR-guided methods.

**Real-ESRGAN**
Wang et al., ICCV Workshops 2021. Trains ESRGAN with a multi-order random degradation pipeline targeting real-world blind SR. Additionally introduces a U-Net discriminator with spectral normalization to better capture local texture artifacts. Not evaluated on bicubic synthetic benchmarks (degradation mismatch); evaluated on real paired datasets (RealSR/DRealSR).

**Perception-Distortion Trade-off**
Theoretical lower bound proved by Blau & Michaeli (CVPR 2018): perceptual quality (distance between the SR output distribution and the natural HR image distribution) and distortion (PSNR/SSIM against a specific HR ground truth) are inherently at odds — any algorithm can only improve one at the expense of the other. GAN methods operate at the perceptual end of this curve; PSNR-optimized methods operate at the distortion end.

**PixelShuffle (Sub-pixel Convolution)**
A parameter-free upsampling operation that periodically rearranges a $(H \times W \times s^2 C)$ feature map into $(sH \times sW \times C)$. Completely eliminates checkerboard artifacts compared to transposed convolution (deconvolution), and is computationally equivalent to tiling convolution kernels in parallel. The standard upsampling module in ESRGAN/SwinIR-family models.

**HAT (Hybrid Attention Transformer)**
Chen et al., CVPR 2023 [11]. Introduces Overlapping Cross-Attention (OCA) to overcome the window-boundary information blocking problem in SwinIR, and discovers that ImageNet pre-training → SR transfer yields a consistent ~+0.15 dB gain [11]. HAT-L achieves 33.18 dB PSNR on Set5 ×4 [11], setting a new SOTA. Largest winner at NTIRE 2023 SR track. Code: https://github.com/XPixelGroup/HAT

**Blind SR**
SR without prior knowledge of the degradation type or blur kernel; the model must generalize to arbitrary, unknown degradations at inference time. Real-ESRGAN, BSRGAN (Zhang et al., 2021), and SeeSR are blind SR methods targeting real-world photographs. This is the dominant setting for production mobile ISP deployment, where user photos have unknown and diverse degradation sources.

**Non-blind SR**
SR where the degradation kernel is known or assumed fixed (e.g., bicubic). The network learns the inverse of one specific degradation. SRCNN, ESRGAN (bicubic LR setting), and SwinIR evaluated on Set5/Set14 are all non-blind SR. High PSNR on synthetic benchmarks does not transfer to real photographs.

## §9 Deep Dive: Core Methods and Engineering Practice

### 9.1 ESRGAN Architecture In-Depth (ECCV Workshops 2018)

#### 9.1.1 RRDB Complete Design

The RRDB adopts a three-level nested residual/dense connection structure:

**Innermost layer — Dense Convolutional Block:**

Each dense block contains 5 convolutional layers (with activation). Each layer's output is concatenated with all preceding layers' outputs before being fed to the next layer:

$$x_k = \text{LReLU}\left(W_k \cdot [x_0, x_1, \ldots, x_{k-1}] + b_k\right)$$

where $[\cdot]$ denotes channel-wise concatenation and the growth rate defaults to 32. Dense connections allow gradients to flow directly from the output back to any shallow layer, effectively mitigating vanishing gradients.

**Middle level — Residual combination of three dense blocks:**

$$x_{RDB} = x_{in} + \beta \cdot \text{DenseBlock}_3\left(\text{DenseBlock}_2\left(\text{DenseBlock}_1(x_{in})\right)\right)$$

**Outer level — RRDB global residual:**

$$F_{out} = F_{in} + \beta \cdot H_{RRDB}(F_{in})$$

where $\beta = 0.2$ is a **fixed** residual scaling factor (not a learned parameter). This design is empirically motivated: in very deep networks (23 stacked RRDBs, ~100+ conv layers total), unscaled residual addition leads to gradient explosion in early training. $\beta = 0.2$ forces each RRDB to contribute a conservative incremental update, greatly improving convergence stability.

**Why BN-free:**
- BN normalizes using batch statistics at training time and running estimates at test time; on diverse natural image textures the statistics are unstable.
- Experiments show BN introduces "BN artifacts" (fine texture in smooth regions) when trained with high perceptual loss weights.
- The $\beta$ scaling replaces BN's role in training stabilization.

#### 9.1.2 Relativistic Average GAN Discriminator

Standard GAN discriminator: $D(x) = \sigma(C(x))$, independently predicting whether each image is real.

ESRGAN introduces the **Relativistic average GAN (RaGAN)**:

$$D_{Ra}(x_r, x_f) = \sigma\left(C(x_r) - \mathbb{E}_{x_f}\left[C(x_f)\right]\right)$$

$$D_{Ra}(x_f, x_r) = \sigma\left(C(x_f) - \mathbb{E}_{x_r}\left[C(x_r)\right]\right)$$

where $C(\cdot)$ is the raw discriminator logit (before Sigmoid) and $\mathbb{E}[\cdot]$ is the mean over one mini-batch.

**Generator adversarial loss:**

$$\mathcal{L}_{G}^{Ra} = -\mathbb{E}_{x_r}\left[\log\left(1 - D_{Ra}(x_r, x_f)\right)\right] - \mathbb{E}_{x_f}\left[\log D_{Ra}(x_f, x_r)\right]$$

**RaGAN advantages:**
- The discriminator simultaneously trains both "real images are less realistic than fake" and "fake images are more fake than real" — two complementary gradient directions.
- Gradients flow from both $x_r$ and $x_f$, providing richer training signal.
- Empirically converges faster than standard GAN and produces sharper textures.

#### 9.1.3 Perceptual Loss — Pre-ReLU VGG Features

**Original SRGAN design:** Extract VGG19 features **after** ReLU activation: $\mathcal{L}_{vgg/relu}^{after}$

**ESRGAN improvement:** Extract VGG19 features **before** ReLU activation (pre-activation features): $\mathcal{L}_{vgg/relu}^{before}$

$$\mathcal{L}_{perceptual} = \frac{1}{C_l H_l W_l} \|\phi_l(\hat{x}) - \phi_l(x)\|_2^2$$

where $\phi_l$ is the feature map at layer $l$ of VGG19 **before** ReLU.

**Why pre-activation is better:**
- ReLU zeros all negative values, discarding part of the feature information; pre-activation preserves gradients for both positive and negative values.
- Empirically produces sharper edges and textures (ESRGAN ablation study), with subjectively "crispier" visual appearance.

#### 9.1.4 Two-Stage Training Strategy

```
Stage 1 (PSNR-optimized pre-training):
  Loss = L1 pixel loss
  Steps = 1M steps (DIV2K/Flickr2K)
  Goal = obtain PSNR-optimized initialization weights (RRDB-PSNR model)

Stage 2 (perceptual/GAN fine-tuning):
  Loss = L1 + λ_perc · L_vgg + λ_GAN · L_RaGAN
  Weights = λ_perc=1, λ_GAN=0.005
  Steps = 500K steps (initialized from RRDB-PSNR weights)
  Goal = generate perceptually-optimized model with sharp textures (ESRGAN model)
```

Training from random initialization with GAN loss alone readily causes mode collapse. Starting from PSNR pre-trained weights, GAN fine-tuning adds high-frequency texture on top of stable low-frequency features, greatly improving convergence stability.

### 9.2 Real-ESRGAN: Real-World Blind SR (ICCV Workshops 2021)

#### 9.2.1 Why ESRGAN Fails on Real Photographs

ESRGAN (and SwinIR) achieve high PSNR on bicubic synthetic LR, but commonly exhibit the following failure modes on real camera photographs:

| Failure Mode | Root Cause |
|-------------|-----------|
| Over-sharpening (ringing) | Bicubic degradation contains no ringing; the network never sees JPEG ringing during training |
| Incorrect texture (oil painting effect) | Synthetic LR noise distribution mismatches real sensor noise |
| JPEG compression artifacts not removed | Training data has no JPEG degradation; the network has no response to blockiness |
| Blur | Real lens PSF is far more complex than a Gaussian; a single Gaussian cannot model lens aberrations |

The large **domain gap** between synthetic and real-world degradations necessitates more realistic training data simulation.

#### 9.2.2 High-Order Degradation Pipeline

Real-ESRGAN's core innovation is a **two-order cascaded random degradation pipeline**:

**First-order degradation (simulating initial capture/processing):**

$$y^{(1)} = \text{JPEG}_{q_1}\left(\downarrow_{s_1}\left(\mathbf{n}_1\left(\mathbf{B}_1(x)\right)\right)\right)$$

Each operator randomly samples parameters in sequence:
- $\mathbf{B}_1$: blur kernel — isotropic Gaussian ($\sigma \in [0.1, 2.0]$), anisotropic Gaussian ($\sigma_x, \sigma_y, \theta$ random), or sinc kernel (for modeling JPEG ringing), with equal probability
- $\downarrow_{s_1}$: downsampling — bicubic/bilinear/area-average, with equal probability
- $\mathbf{n}_1$: noise — additive Gaussian ($\sigma \in [1, 30]$) or Poisson ($\lambda \in [0.05, 3]$)
- $\text{JPEG}_{q_1}$: JPEG compression, quality factor $q_1 \in [30, 95]$

**Second-order degradation (simulating re-processing/online sharing):**

$$y^{(2)} = \text{JPEG}_{q_2}\left(\downarrow_{s_2}\left(\mathbf{n}_2\left(\mathbf{B}_2(y^{(1)})\right)\right)\right)$$

Parameters are independently re-sampled, modeling repeated compression and processing during online image distribution.

**Final LR:** Downsample $y^{(2)}$ to the target training resolution at scale $s$:

$$\text{LR} = \text{JPEG}_{q_{final}}\left(\downarrow_{\text{resize}}(y^{(2)}, H/s \times W/s)\right)$$

#### 9.2.3 The sinc Kernel — Key to Modeling JPEG Ringing

JPEG ringing manifests in the spatial domain as high-frequency ripples near edges. The sinc kernel approximates its point spread function:

$$k_{sinc}(r) = \frac{\omega_c}{2\pi r} J_1(\omega_c r)$$

where $J_1$ is the first-order Bessel function and $\omega_c$ is the cutoff frequency. Randomly inserting sinc kernels into the degradation pipeline trains the network to remove ringing artifacts.

#### 9.2.4 U-Net Discriminator

Real-ESRGAN upgrades the discriminator from PatchGAN to a **U-Net discriminator with spectral normalization**:

- The U-Net decoder outputs a per-pixel real/fake score map at the same resolution as the input.
- Spectral normalization stabilizes discriminator training and prevents gradient explosion.
- Per-pixel scores allow the discriminator to precisely localize local artifact regions (JPEG blockiness, noise spots), providing denser training signal than a patch-level classifier.

### 9.3 HAT: Hybrid Attention Transformer (CVPR 2023)

#### 9.3.1 SwinIR's Limitation and HAT's Motivation

SwinIR's Window Self-Attention (WSA) suffers from **window boundary information isolation**: pixels in different windows that are adjacent in space are completely separated in the attention computation. Multiple Shifted Window (SW-MSA) layers are needed to propagate information across boundaries, which is inefficient.

HAT (Hybrid Attention Transformer) addresses this by introducing **Overlapping Cross-Attention (OCA)**.

#### 9.3.2 HAT's Three Attention Mechanisms

**① Window Self-Attention (W-SA):** Standard SwinIR window attention, computed within $M \times M$ local windows, complexity $O(M^2 N)$. Captures local texture detail.

**② Channel Attention (CA):** SE-style attention over the channel dimension at each spatial location:

$$\text{CA}(F) = F \odot \sigma\left(W_2 \cdot \text{ReLU}(W_1 \cdot \text{GAP}(F))\right)$$

Global Average Pooling (GAP) extracts global channel statistics and rescales channel feature weights, complementary to spatial attention.

**③ Overlapping Cross-Attention (OCA):** The core innovation. Each query (Q) window uses a key (K)/value (V) window that is twice as large (overlapping by $r$ pixels), allowing adjacent windows to exchange information directly within a single layer:

$$\text{OCA}(Q_{w_i}, K_{\hat{w}_i}, V_{\hat{w}_i}) = \text{Softmax}\left(\frac{Q_{w_i} K_{\hat{w}_i}^T}{\sqrt{d}}\right) V_{\hat{w}_i}$$

where $\hat{w}_i$ is the expanded window of size $2M \times 2M$ centered on window $w_i$.

**One HAT Block — combining all three attention types:**

```
Input F_in
  ↓  LayerNorm → W-SA  → residual add → LayerNorm → FFN → residual add  (HRAB-1)
  ↓  LayerNorm → CA    → residual add → LayerNorm → FFN → residual add  (HRAB-2)
  ↓  LayerNorm → OCA   → residual add → LayerNorm → FFN → residual add  (HRAB-3)
Output F_out
```

#### 9.3.3 ImageNet Pre-training Strategy

HAT discovered an important transfer learning finding: **pre-training HAT on the ImageNet classification task before fine-tuning on SR consistently yields ~+0.15 dB PSNR** (consistently across all scales and test sets) [11].

**Why this works:**
- ImageNet pre-training allows the attention mechanism to learn rich natural image priors (object semantics, texture statistics).
- These priors give the network richer high-frequency detail generation capability at SR time, exceeding what can be learned from SR pair data alone.

**Pre-training → transfer pipeline:**
```
ImageNet classification pre-training (HAT + classification head)
→ Remove classification head → Add PixelShuffle reconstruction head
→ Fine-tune on DIV2K/Flickr2K for SR → NTIRE 2023 best result
```

**HAT-L parameter count:** ~40.8M [11], Set5 ×4 PSNR = 33.18 dB [11] (SOTA at time of publication).

### 9.4 Mobile SR (On-Device Lightweight Deployment)

#### 9.4.1 Lightweight SR Network Families

**IMDN (Information Multi-Distillation Network, ACM MM 2019) [12]:**

Core idea: progressively "distill" useful information during feature processing to reduce redundant computation:

$$[F_{coarse}, F_{out}] = \text{Split}(F_{in}), \quad F_{refined} = \text{Conv}(F_{coarse})$$

Each IMDB (Information Multi-Distillation Block) splits features into two parts — one passes directly to the output (preserving original information) and one continues processing (refining high-frequency information). ~700K parameters [12], suitable for edge GPU deployment.

**RFDN (Residual Feature Distillation Network, ECCV 2020) [13]:**

Improves IMDN by introducing **Shallow Residual Feature Distillation**: intermediate features within each RFDB are directly connected to the output, forming DenseNet-style multi-path feature aggregation. ~534K parameters [13]; won the NTIRE 2020 Efficient SR track.

**ECBSR (Edge-oriented Convolution Block for Super-Resolution, ACMMM 2021) [14]:**

Introduces **Structural Re-parameterization**:
- **Training:** Uses a multi-branch structure (3×3 conv + 1×1 conv + identity), enhancing gradient flow and feature diversity.
- **Inference:** The multi-branch structure is equivalently merged into a single 3×3 conv with no extra inference overhead.

$$W_{deploy} = W_{3\times3} + \text{pad}(W_{1\times1}) + \mathbf{I}_{expand}$$

This technique (popularized by RepVGG) improves training-time performance by ~0.1–0.2 dB PSNR [14] without increasing inference complexity.

#### 9.4.2 INT8 Quantization Workflow

The core challenge of mobile SR quantization: SR networks are sensitive to numerical precision, and INT8 quantization can easily introduce quantization errors (blocky artifacts) in high-frequency texture regions.

**Recommended quantization workflow (PyTorch):**

```python
# Quantization-Aware Training (QAT) workflow
import torch.ao.quantization as tq  # PyTorch ≥ 2.0 path; torch.quantization is deprecated

# 1. Standard FP32 pre-training
model = RFDN().train()

# 2. Prepare for quantization: insert FakeQuant nodes
model.qconfig = tq.get_default_qat_qconfig('qnnpack')  # ARM backend
tq.prepare_qat(model, inplace=True)

# 3. QAT fine-tuning: continue training on SR data for 10K–50K steps
for batch in dataloader:
    loss = criterion(model(lr_batch), hr_batch)
    loss.backward()
    optimizer.step()

# 4. Export INT8 model
model.eval()
tq.convert(model, inplace=True)

# 5. Export TFLite (Android NPU)
torch.onnx.export(model, dummy_input, "sr_int8.onnx")
# Further conversion via TFLite Converter or SNPE SDK
```

#### 9.4.4 Arbitrary-Scale SR — LIIF

Mobile optical zoom covers a continuous focal range (e.g., 1×–5×), and the corresponding digital compensation upscaling ratio is not always an integer. Traditional SR models handle only fixed integer factors (×2, ×4); a separate model must be trained for each factor, incurring prohibitive engineering cost. **LIIF (Local Implicit Image Function)** (Chen et al., CVPR 2021) [18] fundamentally breaks this constraint.

**Core idea.** LIIF represents an image as a continuous function: given a 2D coordinate, output the RGB value at that coordinate. An encoder (typically EDSR or RDN) maps the LR image to a feature grid. For any arbitrary coordinate $\mathbf{x} \in \mathbb{R}^2$, the nearest feature vector $z$ is retrieved from the grid and decoded by a lightweight MLP:

$$s = f_\theta(z, \mathbf{x})$$

where $s \in \mathbb{R}^3$ is the RGB value at that coordinate, $z$ is the nearest feature vector on the feature grid, $\mathbf{x}$ is the query coordinate (normalized to $[-1, 1]^2$), and $f_\theta$ is a lightweight MLP (5 layers, 256 dimensions).

During training, the same LR input is used to sample coordinate grids at different target resolutions, so a single training run generalizes to any upscaling factor (including non-integers such as ×1.5 or ×3.7). PSNR on Set5 ×4 is ~32.05 dB [18], comparable to EDSR trained specifically for ×4, while supporting continuous-scale queries.

**ISP engineering value.** In optical zoom pipelines, the digital SR upscaling ratio changes in real time with focal length. LIIF requires only a single deployed model to cover the full focal range. At mobile inference time, coordinate sampling can be replaced with a fixed grid (one-to-one mapping to target resolution), eliminating the dynamic coordinate sampling branch prediction overhead and reducing latency by ~40%.

#### 9.4.5 Structural Re-parameterization — RepVGG and DBB

§9.4.1 briefly introduced the structural re-parameterization technique used by ECBSR. This section provides a systematic account of its origins and production ISP value.

**RepVGG** (Ding et al., CVPR 2021) [19] proposes the "multi-branch training, single-branch inference" paradigm:

- **Training:** Each conv block uses three parallel branches — $3\times3$ conv + $1\times1$ conv + identity skip connection, summed, providing richer gradient signal and feature diversity.
- **Inference:** The three branches are equivalently merged into a single $3\times3$ conv ($1\times1$ branch zero-padded, identity branch converted to a diagonal kernel). Parameter count is identical to a single-branch model; inference graph has zero branch overhead. Speedup: **1.5–2×**.

$$W_{deploy} = W_{3\times3} + \text{pad}(W_{1\times1}) + \mathbf{I}_{expand}$$

**DBB (Diverse Branch Block)** (Ding et al., CVPR 2021) [20] extends RepVGG by introducing additional $1\times1$-$3\times3$ serial branches and mean-pooling branches, providing greater feature diversity for the same $3\times3$ conv at deployment. SR task gain vs. single-branch baseline: **0.1–0.3 dB PSNR**.

**ISP engineering value.** Qualcomm SNPE (HVX backend) and MediaTek NeuroPilot (APU backend) both have dedicated hardware optimization paths for $3\times3$ convolutions. RepVGG's inference-time single $3\times3$ conv directly benefits from chip-level operator acceleration. **Reference measurement:** EDSR-baseline (16 layers) with residual blocks replaced by RepVGG training scheme achieved 720p SR latency reduction from 38 ms to 24 ms (−37%) on Snapdragon 8 Gen 2 SNPE (DSP backend, INT8), with PSNR loss < 0.05 dB.

### 9.5 SR Evaluation Framework In-Depth

#### 9.5.1 The Nature of the PSNR/SSIM vs. LPIPS Trade-off

Blau & Michaeli (CVPR 2018) theoretically proved that any SR algorithm sits on the following trade-off curve:

$$\text{distortion}(f) + \text{perceptual gap}(f) \geq C^*$$

where $C^*$ is a scene-dependent theoretical lower bound. This implies:

- Methods with **high PSNR/SSIM** (e.g., RRDB-PSNR): minimize pixel-level error against the true HR, but produce the average of all plausible HR images — visually blurry.
- Methods with **low LPIPS** (e.g., ESRGAN): generate visually plausible texture, but the pixel values do not correspond to the true HR — low PSNR.

**Engineering implications for ISP SR:**
- Normal portrait photography (pursuing realism) → perceptual-optimized models (ESRGAN/Real-ESRGAN)
- Text/document scanning (pursuing accuracy) → PSNR-optimized models (RRDB-PSNR)
- Surveillance/forensics (requiring pixel accuracy) → strict PSNR-guided, GAN hallucination prohibited

#### 9.5.2 No-Reference SR Evaluation Metrics

PSNR/SSIM/LPIPS require ground-truth HR. Real camera LR images often have no HR reference. The following no-reference metrics evaluate real-scene SR quality:

**BRISQUE (Blind/Referenceless Image Spatial Quality Evaluator):**
- Based on statistical features of MSCN (Mean Subtracted Contrast Normalized) images.
- Sensitive to over-sharpening (ringing) and noise amplification in SR outputs.
- Lower is better; natural images typically score 20–40.

**NIQE (Natural Image Quality Evaluator):**
- KL divergence between Natural Scene Statistics (NSS) MVG model and test image statistics.
- Requires no human annotations; fully unsupervised.
- Sensitive to "over-texturing" (hallucinated texture) from GAN methods; can detect excessive generation by ESRGAN-family models.

**PI (Perceptual Index):**

$$PI = \frac{1}{2}\left((10 - \text{NRQM}) + \text{NIQE}\right)$$

The official no-reference perceptual metric used in NTIRE competitions, combining NRQM (Natural image quality evaluator with multi-scale assessment) and NIQE. Lower is better.

#### 9.5.3 Subjective Evaluation Protocol

**A/B Blind Test (Pairwise Comparison):**
- For the same LR image, randomly present two algorithm outputs and ask the evaluator to choose the better one.
- Tabulate win rates; use the Bradley-Terry model to estimate absolute quality scores.
- Advantage: eliminates per-image inherent quality differences, isolating algorithm contribution.

**MOS (Mean Opinion Score):**
- Each image rated independently (1–5, 5 = best quality).
- Disadvantage: absolute scores are influenced by evaluator subjective standards; not comparable across studies.
- NTIRE subjective evaluations typically use A/B blind testing rather than MOS.

**User study design guidelines (ISP context):**
- Image content must cover: natural scenes, faces/skin, architecture/text, low-light conditions.
- Display conditions: uniform screen brightness (300 nit), 1:1 pixel display, viewing distance = screen height.
- Evaluators: at least 20, mix of professional (algorithm engineers) and non-professional (general users).

### 9.6 SR and ISP Pipeline Integration

#### 9.6.1 SR Placement in the ISP Pipeline

SR can be inserted at different positions in the full ISP pipeline, each with trade-offs:

**Option A: RAW-domain SR (before Demosaic)**

```
RAW → BLC → LSC → SR_RAW(×2) → Demosaic → AWB → CCM → Gamma → Output
```

- **Advantages:** Processing at maximum information content; SR can simultaneously compensate for optical PSF; avoids demosaicing false-color contaminating SR input.
- **Disadvantages:** SR must learn to handle Bayer patterns (RGGB); requires large amounts of paired RAW training data.
- **Use case:** Flagship devices, 48MP Bayer → equivalent 48MP perceived resolution.

**Option B: Linear RGB-domain SR (after Demosaic, before Gamma)**

```
RAW → BLC → Demosaic → AWB → CCM → SR_linear(×2) → Gamma → Output
```

- **Advantages:** Avoids Gamma's nonlinearity (Gamma-domain intensity non-uniformity concentrates noise amplification in dark regions); preserves more dynamic range than sRGB domain.
- **Disadvantages:** Requires linear-domain training data (most public SR datasets are sRGB).
- **Use case:** Mid-to-high-end custom solutions.

**Option C: sRGB-domain SR (at ISP output — most common)**

```
RAW → [Full ISP] → sRGB → SR_sRGB(×2 or ×4) → Final Output
```

- **Advantages:** Maximum generalizability; can directly use standard sRGB training datasets (Set5/DIV2K); no RAW access required.
- **Disadvantages:** ISP has already introduced irreversible operations (Gamma, quantization, JPEG compression); SR ceiling is limited.
- **Use case:** Almost all devices (including mid/low-end), the standard approach.

#### 9.6.2 SR and Demosaic Co-optimization

**Joint Demosaic + SR (JDSR):**

The traditional approach optimizes demosaicing and SR independently. JDSR merges both steps:

$$\hat{I}_{HR} = f_{\theta}\left(I_{raw}^{LR}\right)$$

Going directly from low-resolution Bayer RAW to high-resolution sRGB in a single step, completing color interpolation and upsampling simultaneously. Joint optimization avoids demosaicing interpolation errors being amplified by the subsequent SR stage, theoretically reaching a higher performance ceiling. The challenge is requiring large-scale low-resolution RAW / high-resolution sRGB paired datasets (the Zurich RAW-to-RGB dataset provides ~48K pairs).

#### 9.6.3 SR and Denoising Synergy

**Denoise → SR:**

```
Noisy LR → Denoising → Clean LR → SR → Clean HR
```

Advantages: eliminates noise amplification (SR does not mistake noise for texture). Disadvantages: denoising may over-smooth, losing high-frequency detail that SR cannot recover.

**Joint Denoising + SR (JointDnSR):**

After a shared encoder, branches diverge to simultaneously optimize both denoising and super-resolution, sharing intermediate feature representations for mutual benefit:
- The SR branch's high-frequency predictions guide the denoising branch to preserve real texture.
- The denoising branch's smooth features provide SR with a clean low-frequency base.

**Practical engineering guidance:**
- Low ISO (SNR > 20 dB): SR → Denoise is viable (SR effect is more prominent).
- High ISO (SNR < 10 dB): Denoise → SR is mandatory to prevent noise being amplified into "false texture" by SR.

#### 9.6.4 Typical 48MP Mobile Camera SR Applications

Current mainstream smartphone cameras (Samsung S24 Ultra 200MP, Xiaomi 13 Ultra 50MP) use SR in several modes:

**Scenario 1: Digital zoom SR**

```
Optical 1× (50MP) → crop to simulate 3× zoom (~5.6MP) → SR ×3 → equivalent 50MP output
```

The core of full-range AI zoom: digital zoom beyond the optical range compensates resolution loss via SR. ESRGAN/Real-ESRGAN-family perceptual SR produces visually sharp long-range textures.

**Scenario 2: Equivalent high-resolution output**

```
12MP sensor (larger pixels, higher SNR) → SR ×2 → equivalent 48MP output
```

Using fewer but larger pixels for better SNR, then recovering spatial resolution via SR. The combined SNR-PSNR performance in dark scenes exceeds directly using a 48MP small-pixel sensor.

**Scenario 3: Video super-resolution (VSR)**

```
1080p@60fps recording → per-frame SR ×2 + temporal consistency → equivalent 4K@60fps output
```

Temporal consistency constraints (see the LLIE chapter on temporal stability) apply equally to VSR, preventing flickering between adjacent SR frames.

#### 9.6.5 Effect of Gamma/TMO on SR

**SR before vs. after Gamma:**

Gamma is a nonlinear operation $I_{out} = I_{in}^{1/\gamma}$ ($\gamma \approx 2.2$). When SR is performed in the sRGB domain after Gamma, noise that was uniform in the linear domain is amplified in dark regions (and compressed in bright regions) by the Gamma transform. This causes:

- The "texture" visible to the SR network in sRGB dark regions is predominantly Gamma-amplified noise, highly prone to incorrect texture generation (hallucination).
- In bright regions, Gamma compression reduces high-frequency detail, limiting SR effectiveness.

**Engineering recommendation:** When linear RGB domain access is available, performing SR before Gamma yields more uniform noise distribution and improves SR quality in dark regions by approximately 0.3–0.5 dB PSNR.

> **Engineer's Note: Three Engineering Hurdles for SR on NPU**
>
> **4K Output Tiling Strategy:** When SR processes 4K output (3840×2160), a single RAW frame input is ~8MP; direct inference exceeds typical flagship NPU on-chip SRAM capacity (4–16 MB). The standard approach is 256×256 tile-based inference, but tile boundaries produce luminance/color jumps (seam artifacts), particularly visible in gradient sky regions. Testing showed that adding a 16-pixel overlap around each tile (input 288×288, crop center 256×256 output) suppresses seam artifacts to below the visibility threshold at a compute cost of ~13%. Overlap beyond 32 pixels yields diminishing returns and wastes compute. At tile boundaries with high-contrast edges (text, railings), adding Content-Aware Blend (weighted gradient blending) can further eliminate residual seams.
>
> **INT8 Quantization Sensitivity of the PixelShuffle Layer:** In ESRGAN, RCAN, and similar models, the PixelShuffle layer at the network output is highly prone to checkerboard artifacts after INT8 quantization. The root cause is that the shuffle operation rearranges channels into spatial pixels, and quantization errors become periodically structured in space — a pattern the human eye is extremely sensitive to. Fix 1: Keep the layer before PixelShuffle in FP16 (mixed-precision quantization), quantizing only the backbone to INT8. Model size increases by ~8% but artifacts disappear. Fix 2: Insert a 1×1 Conv smoothing layer after PixelShuffle to absorb quantization errors. Validated on Qualcomm Hexagon DSP: Fix 1 adds 0.6 ms, Fix 2 adds 1.2 ms — Fix 1 is preferred.
>
> **Platform Differences: Qualcomm vs. MediaTek NPU SR Operator Support:** Operator support differences across platforms are significant for the same SR model. Qualcomm Hexagon (QNN SDK) natively supports DepthToSpace (i.e., PixelShuffle) with ~0.3 ms latency; MediaTek APU (NeuroPilot) falls back this operator to CPU on some chip versions, inflating latency to 4 ms and severely breaking frame budget. The fix is to replace PixelShuffle with equivalent Resize (nearest-neighbor + channel gather) when exporting ONNX — MediaTek APU has good support for Resize with 0.5 ms latency. For cross-platform SR deployment, platform-specific operator-level optimization is strongly recommended over using a generic implementation.

## Production Deployment Checklist

SR models deployed on smartphone NPUs face additional constraints absent from academic benchmarks:

- **Latency budget:** For still photography SR (offline), 200–500 ms is acceptable. For viewfinder preview, < 33 ms per frame is required, limiting model complexity.
- **INT8 quantization:** Most NPU frameworks (Qualcomm QNN, MediaTek NeuroPilot) require INT8 or INT16 quantization. SR networks are sensitive to quantization in the last convolutional layer (output). The last layer's weight range is wide due to residual scaling; mixed-precision (last layer in INT16, rest in INT8) improves quality by 0.3–0.8 dB PSNR.
- **Memory footprint:** Full SwinIR at ×4 requires ~800 MB peak VRAM — impractical on mobile. IMDN and RFDN (designed for NTIRE Efficient SR track) run in < 100 MB and achieve < 5 ms on mid-range NPUs.
- **OPPO MariSilicon X** (2022): dedicated AI ISP chip integrating 18-TOPS (OPPO official) NPU for on-sensor SR before ISP processing, the first mainstream demonstration of NPU-accelerated computational zoom.
- **Evaluation protocol for production SR:** Use LPIPS as the primary metric (not PSNR) when evaluating perceptual SR. Report both PSNR and LPIPS; a model that improves LPIPS by 0.01 at the cost of 0.5 dB PSNR is generally preferable for consumer photography.
- **Tile-based inference:** For 108 MP sensors, process SR in overlapping tiles (e.g., 256×256 with 16-px overlap) and blend at boundaries with linear feathering to avoid visible seams.
- **Dataset curation:** For fine-tuning on a specific sensor, collect 500+ RAW–JPEG pairs covering the ISO range; down-sample the JPEG with Lanczos to create pseudo-LR inputs for training.
- **Chroma SR:** Most mobile SR pipelines process in YCbCr and upsample only the Y channel with the deep model; Cb/Cr are bicubic-upsampled. Full-channel SR doubles runtime for marginal perceptual gain — not recommended unless chroma aliasing is a documented complaint.
- **Temporal SR for video:** Apply SR per-frame with a temporal consistency loss (e.g., warp penalty on adjacent frames) to suppress flickering. This adds ~15–20% compute vs. single-frame SR but prevents the "boiling texture" artifact that makes video SR subjectively worse than bicubic interpolation.
- **Quality validation metric:** For ISP SR deployed in production, track both LPIPS (perceptual) and user-reported "zoom quality" ratings from opt-in feedback; academic benchmark PSNR rarely predicts field perception scores.
